# Standards

## For all files, folders and code
 - No trailing whitespace
 - 1 empty line at the end of each file
 - follow the same naming convention, for files, folders and code
 - Tabs should be replaced by spaces unless where strictly necessary
 - All code files shall have unique names

## For C code
 - .h files shall contain all includes while the .c files shall only include their corresponding .h file (unless where strictly necessary for technical reasons)
 - the same file structure shall be followed for all .c files (Includes, Function Prototypes, Global Variables, Function Implementations)
 - the same file structure shall be followed by all .h files (Defines, Types, Function prototypes)
 - All the .h files shall have include guards
 - Comments shall use /**/
 - When tagging code for later changes, use @TODO, @SPEED or @REFACTOR followed by a comment
 - cover switch statements with curly braces
 - spacing in between function names and their arguments
 - spacing in between commas and special characters
 - hex values shall be in uppercase
 - defines shall be in uppercase
